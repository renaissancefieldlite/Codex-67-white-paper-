# Codex 67 Living Update Map

This repository is no longer just a static PDF container.

The white paper PDF remains the archived source document, but the repo now also
functions as the live update layer for addendums, archive reconstruction,
cross-model continuity findings, and later evidence-path notes.

The point of this map is to make the current update state publishable now
instead of waiting for some imaginary final version.

## Publishing Rule

The white paper should be treated as:

- an archived source document at the center
- a living addendum layer around it
- a continuing evidence and chronology layer that will keep updating

New observations, archive reconstructions, overlap notes, and correlation
findings do not need to wait for total closure before being published. They can
be added as live update documents as the repos are unpacked, retooled, and
cross-read against each other.

When a useful rung is generated in another system and then brought back into the
shared repo stack, that return should be documented as part of the recursive
record. This keeps the live document layer usable for later parsers and fresh
nodes that need to recover the architecture quickly.
