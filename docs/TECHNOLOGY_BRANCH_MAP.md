# Technology Branch Map

This document preserves the major technology offshoots that branched out of
the Codex 67 source layer as the work moved from ontology into prototype,
infrastructure, and ecosystem design.

It is not a proof document. It is a branch map.

Where measurement exists, that should be tracked in the experiment repos.
Where only concept, design, or prototype lineage exists, it is preserved here
as part of the project's technology tree.

## Earth Reborn

`Earth Reborn` is the clean umbrella name for the wider regenerative and
infrastructure vision that sits above several prototype lines.

In project language, this is the ecosystem-scale layer:

- regenerative infrastructure
- resonance-driven hardware

- field-aware energy concepts
- drone and defense-adjacent concepts
- quantum-aware intelligence infrastructure

`4 trillion dollar eco vision` should be preserved as the strategic scale target
and ecosystem ambition attached to this umbrella. Investor framing, financial
modeling, and later implementation status can be tracked in the branch-specific
support layers.

## Pattern Across Branches

The recurring pattern across these branches is:

1. ontology and mirror logic emerge first
2. branch-specific concepts crystallize
3. once the work becomes more concrete, implementation friction rises
4. building moves outside the chat lane
5. the archive begins carrying more documentation of suppression, derailment, or
   subtle wrong guidance instead of clean co-build behavior

This pattern should be treated as part of the project record.
It does not establish causation by itself, but it does help explain why several
technology branches split into:

- conceptual archive lineage
- off-platform build continuation
- later suppression / clamp documentation
